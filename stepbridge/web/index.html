User Guide

StepBridge is a CV/Pulse/MIDI step sequencer for the Music Thing Modular Workshop Computer.

Panel controls

  • Down (Utility): Main knob = sequence length, Y knob = track select.
  • Middle (Step Edit): Main knob = pitch of the scrubbed step (turn fully down for rest), X knob = step scrub, Y knob = track select.
  • Up (Tempo): Main knob = tempo (only while using the internal clock).

Clock

Patch a clock signal into Pulse In 1 to sync to an external clock; leave it unpatched to use the internal tempo knob. Pulse In 2 resets all tracks to step 1.

Pressing Play arms the sequencer rather than guaranteeing an instant start. With External Pulse or MIDI Clock selected, playback genuinely can't begin until the next real pulse/tick arrives from the external clock - there's no way to predict or control exactly when that will be relative to when you pressed Play, since the external clock keeps running on its own regardless of our transport state. This is standard behavior for any instrument that follows an external clock, not a bug: you'll see the transport label show "Armed…" (pulsing) until the first pulse actually arrives and playback truly starts. Once that first pulse lands, every step after it is correctly locked to the external clock's actual rate - only that first moment is variable.

Web UI

Click a step in the Sequencer tab to set its pitch, or click near the bottom of a step to make it a rest. Use the transport buttons to stop, pause, or play the sequencer from the browser.

Pull model: every request gets either a data response or an explicit NACK; a request with no reply within 750ms is logged as a timeout in the connection diagnostics panel. MSG_PLAYHEAD is the only unprompted broadcast, sent only when the playhead actually changes.

Note: every SysEx data byte, including the command byte, must be 0-127. Host→Firmware IDs were originally specced at 0x60-0x89 and renumbered to 0x50-0x79 after this was caught during milestone-3 hardware testing (0x85 is 133, an illegal MIDI status byte value).

MIDI Clock: firmware honors incoming realtime Start(0xFA)/Continue(0xFB)/Stop(0xFC) as transport commands and 0xF8 (24 PPQN) for step timing, but only once MIDI Clock is the explicitly selected source - it never overrides Internal/ExternalPulse on its own. Selecting MIDI Clock from the Web UI is sticky until explicitly changed back; patching/unpatching Pulse In 1 does not affect it.
